Create a Table

This article describes how you can create a new table in ONE Data. You can choose between the following options:

  • Create empty table: Start from scratch and define the table structure and attribute values directly in ONE Data.

  • Import from Catalog Item: Use an existing catalog item from the Knowledge Catalog. This is especially useful in these cases:

    • You have a data source with ungoverned data that you can’t manage otherwise.

    • You need to quickly modify some data before exporting it back to the data source.

    • The catalog item contains data that can be used as reference data.

  • Upload file: Upload a local file from your machine. Currently, you can import files in CSV, TSV, XLS, and XLSX formats.

Creating a new table means making the data discoverable in ONE Data. Once your data is loaded, you can enhance it by adding or removing attributes or modifying the table values.

Note

ONE Data is a type of data source in ONE and you can access the metadata of each ONE Data table in the Knowledge Catalog as well. These tables are labeled as ONE Data catalog items.

To easily navigate to ONE Data from the Knowledge Catalog, when viewing the table in the Knowledge Catalog, go to Data > Open in ONE Data.

Create empty table

To use ONE Data as your table creation tool, follow these steps.

  1. In ONE Data, select Create table.

  2. Choose Create empty table and select Continue.

  3. Enter a name for the table and select Confirm.

    For example, if you want to create a table of all events that your marketing team organized over the past year, let’s name the table Events.

  4. You’re now redirected to the table overview and can start defining the attribute names and data types. For each attribute that you want to add, do the following:

    1. In the column header, select the Plus icon (+).

    2. Specify the attribute name and data type. For example, add an Event name attribute of type String.

      Caution
      		 Keep in mind that you cannot change the data type once the attribute is created. In this case, you would have to delete it and create a new one with the correct type.

    3. Select Confirm to create the attribute.

    4. Repeat steps a-c for each attribute that you want to add.

      For our Events table, we will define three more attributes: Date (of type Date), Location (of type String), No of attendees (of type Integer).

  5. Once the table structure is ready, you can fill in your records.

    1. To create a new row, in the row header select the Plus icon (+).

    2. When it comes to inputting data, there are several options.

      • To fill in a single cell: Select a cell and start typing. Your changes are saved automatically as soon as you press Enter or select another cell.

      • To fill in the entire column at once: If the same value should be used in multiple rows, right-click the column name and select Multi-edit values. Provide the value and select Apply. This value is then inserted to all rows in the column.

        For example, if most of our events are held in the same city, we could use this option to specify the location for all events and then edit any outliers individually.

  6. Your table is now ready for use. See the Next steps section for more tips about how to proceed.

Import from catalog item

Important

This option is not supported for catalog items with attributes of binary data type.

For more general information about data export, see data-export.html.

To load data from an existing catalog item, do the following:

  1. In ONE Data, select Create table.

  2. Choose Import from Catalog item and select Continue.

  3. Find and select the catalog item that you want to use. Use full-text search and filters as needed.

    Tip
    		 You can filter by assigned or suggested glossary terms, DQ percentage, data source or location, number of catalog item attributes or records, last profiling date, catalog item owner, or detected anomalies. In addition, you can also view only published catalog items or drafts.

  4. Once you select the catalog item, select which attributes the new ONE Data table should contain.

    • To import all attributes: Select the checkbox in the column header.

    • To import a selection of attributes: Select the attributes individually. You can narrow down the list using the full-text search and sort by the attribute name, data type, comments, or description.

      Tip

      • Try searching for integer or string to filter attributes by their data type.

      • Unsure about what data you’re working with? Switch to the Data Preview tab to see a live preview of the top 50 records in the catalog item. You can also select attributes directly from this tab. Keep in mind that the preview isn’t available for virtual catalog items.

      • If you have multiple attributes with similar data and the quality overview on the Attributes tab isn’t sufficient, check out their detailed profiling and DQ results on the Profile & DQ Insights tab. You can both filter and select attributes from here, there’s no need to switch back and forth between tabs. Keep in mind that this information is available only for data that has been profiled.

  5. Once you choose the attributes, select Next.

  6. Enter a unique name for the table and optionally a description.

  7. Select Create table. Depending on the size of your catalog item, it might take a few minutes to get everything ready. To continue working with the platform in the meantime, select Run in background. This takes you to the newly created ONE Data catalog item in the Knowledge Catalog.

    A notification lets you know when the import is finished. Alternatively, remain on the same page until your ONE Data table is created.

  8. Your table is now ready for use. See the Next steps section for more tips about how to proceed.

Upload file

To import a local file, follow these steps:

  1. In ONE Data, select Create table.

  2. Choose Upload file and select Continue.

  3. Drag and drop a file or select Browse files. You can import only one file at a time.

    The following formats are supported: CSV, TSV, XLS, and XLSX. Currently, only the first sheet in Microsoft Excel files is imported.

    Tip
    		 A validation check is performed before any data is loaded, preventing you from importing files with incorrect formatting. If the file upload fails, check the file structure for validity issues in another text editor.
    Caution
    		 Keep in mind that importing large Excel files (with tens of thousands of records or more) will temporarily affect the application performance. We therefore recommend working primarily with files with a lower number of records.

  4. Check the file preview to make sure the data was read correctly. The first valid 50 records are displayed; any invalid records are hidden from the preview and not imported.

    • To edit the attribute name and data type: Double-click the column header and change the values as needed, then select Confirm.

      Keep in mind that while you can select any data type for any attribute, records will be labeled as invalid and omitted during the import if the parsing results don’t match what you selected.

    • To change the automatically detected file options: As needed, modify the detected file format, value and decimal separators, date and date time formats, and file encoding.

    • To use the first row as the header: Select Use first row as headers.

      Caution
      		 Currently, this option doesn’t work as expected and is reported as a known issue.

    • To import a selection of columns: By default, all columns are selected for import. To enable this option, click Select specific columns to import (or, use the keyboard shortcut) and make your selection.

      Tip
      		 While preparing the file for upload, you can also remove it and upload another one instead. If you prefer to go back to the default settings, select Reset settings.

  5. Once you’re happy with your choice, select Continue.

  6. Enter a unique name for the table and optionally a description.

  7. Select Create ONE Data table. Depending on the size of your file, it might take a few minutes to get everything ready. To continue working with the platform in the meantime, select Run in background. This redirects you to the All Tables tab.

    A notification lets you know when the import is finished. Alternatively, remain on the same page until your ONE Data table is created.

  8. Your table is now ready for use. See the Next steps section for more tips about how to proceed.

Next steps

Once you have successfully created a ONE Data table, start exploring what ONE Data and ONE can offer you.

  • Change the structure or the contents of your table at any point from the Data tab of your table, as explained in Get Oriented with ONE Data, sections Edit table model and Edit data inline.

  • Check the data quality of your table (dq-evaluation-in-one.html) or learn how to use it in a rule (Validate Data using ONE Data Tables).

  • If you want to export the data back to the origin data source, see Get Oriented with ONE Data, section Export data to another data source.

  • Or, share the table with your team members and collaborate with them using tasks and comments. See Get Oriented with ONE Data, sections Permissions and access and Tasks and comments.
